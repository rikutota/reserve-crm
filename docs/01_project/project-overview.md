# Reserve CRM プロジェクト概要書

## 1. プロジェクト名

Reserve CRM

## 2. 概要

Reserve CRM は、小規模サロン向けの予約・顧客・請求管理システムである。

予約情報、顧客情報、請求・売上情報を一元管理することで、予約ミスの防止、顧客情報検索の効率化、売上集計作業の削減を目的とする。

## 3. 対象顧客

1店舗で運営される小規模美容サロンを対象とする。

想定する店舗規模は以下の通り。

- スタッフ数：3名程度
- 月間予約数：約200件
- 予約経路：電話、LINE、Instagram DM、店頭
- 現在の管理方法：紙の予約表、Excel、手作業集計

## 4. 現状の課題

対象顧客は、予約・顧客・売上情報を複数のツールで分散管理している。

その結果、以下の課題が発生している。

- 予約情報の転記ミスが発生する
- 予約の重複に気づきにくい
- 顧客情報を探すのに時間がかかる
- 来店履歴を確認しづらい
- 月末の売上集計に時間がかかる
- スタッフ別・メニュー別の売上を把握しづらい

## 5. プロダクトゴール

スタッフが、顧客登録、予約登録、来店後の請求、売上確認までを1つのシステムで行える状態にする。

特に、以下の実現を目指す。

- 予約重複ミスを防止する
- 顧客情報検索を簡単にする
- 会計・売上管理を一元化する
- 日次・月次の売上確認を効率化する

## 6. 開発方針

本プロジェクトは、アジャイル寄りのスクラム風開発で進める。

ただし、実務開発を想定し、最低限の上流工程成果物を作成する。

開発は1週間スプリントで進め、各スプリントごとに以下を行う。

- スプリントゴールの設定
- バックログの選定
- 実装
- テスト
- スプリントレビュー
- レトロスペクティブ
- 次スプリントへの改善反映

## 7. MVP方針

初回リリースでは、顧客登録から予約、来店、請求、売上確認までの最小業務フローを実現する。

MVPでは以下を対象とする。

- ログイン
- スタッフ管理
- メニュー管理
- 顧客管理
- 予約管理
- 予約重複チェック
- 簡易請求
- 支払い済み登録
- 日次売上一覧

## 8. MVP対象外

初回リリースでは、以下は対象外とする。

- LINE連携
- Web予約フォーム
- クレジットカード決済
- 請求書PDF発行
- 複数店舗対応
- 会計ソフト連携
- 顧客向けマイページ
- 回数券管理
- キャンセル待ち機能

## 9. 成功条件

MVPの成功条件は以下とする。

- スタッフが顧客を登録できる
- スタッフが予約を登録できる
- 同じスタッフの同時間帯予約を防止できる
- 来店後に請求を作成できる
- 支払い済みにできる
- 日次売上を確認できる
- 基本操作をマニュアルなしでも理解できる

## 10. 想定リリース

MVPを約8〜10スプリントで完成させ、本番想定環境へリリースする。

リリース後は、運用フィードバックをもとに改善バックログを作成し、追加開発を検討する。
